# Update 2026.07.16

1. Creator Studio co-writing got a big upgrade: generation no longer fears interruptions — job progress is saved, and even a service restart picks up where it left off; you can watch progress, finished sections are delivered as they complete, and you get a notification when it's done. Finished stories export to Markdown, TXT, or EPUB, and can become share cards; a new bookshelf displays your works in one place, and the version panel lets you preview and restore older drafts in one click.
2. Co-written stories are easier to start: one character is enough (a second is optional). The completion page now offers "what's next" exits — continue straight into a side story, or start a new one from a memory or a moment; starter templates lower the bar when you don't know where to begin. Before writing, a material-richness indicator shows whether there's enough source material, with a gentle nudge (never a blocker) when it's thin.
3. The provider "Test" button now actually tests: it connects to your configured service and reports check by check — LLMs list their models and run a one-token exchange, embedding models verify dimension compatibility, TTS lists voices or synthesizes a short sample, search runs a real query, and image/video services confirm the endpoint is reachable. A wrong API key, a bad URL, or a nonexistent model name shows up within seconds of pressing Test — not mid-conversation.
4. Image providers gained a "Deep test": it really generates one small image to verify the whole generation chain end to end (this incurs a tiny API cost, and asks for confirmation first).
5. Provider settings fixes: the Base URL field is back — a regression in the previous release removed it, making local/custom APIs unconfigurable, breaking "fetch models", and sometimes silently wiping a saved URL when editing an existing connection. Built-in presets (NanoGPT, OpenRouter, …) now work with the field left blank, using their default endpoints.
6. No more guessing the SearXNG setup: a persistent hint next to the field now explains you only need the instance root (e.g. `https://searxng.example.com`) — the app builds the query path itself — and reminds you the instance must enable the json format in settings.yml. Previously saved SearXNG connections keep working and convert automatically on their next save.
7. When image generation hits a storage outage, you no longer get an inscrutable system error: the app now says plainly that the media object storage is unreachable and what to check, the server logs record the full cause, and retrying no longer loses already-generated candidates.
8. Sending photos to characters reads more naturally: fixed an issue where, with a text-only chat model plus a separate vision model, characters could mistake "some small text in the image is unreadable" for "your message is hard to read". Characters now also keep straight which image you just sent, which came earlier, and which they generated themselves.
9. Usage and cost reports now have their entry right in the admin menu and home page — no more typing the URL by hand.
10. The scene-presence hint can be turned off: if you'd rather not see the "A message feels more natural first" strip when opening a chat, switch it off under Settings → Personal → Scene presence hint. When off, it only appears when you actively try the shared scene yourself (all its actions remain available).
11. A small build-from-source fix: `docker compose build` no longer intermittently fails with `image already exists` (most likely on a first-clone cold build).
