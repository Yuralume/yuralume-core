# 2026.07.16 更新版本

1. 創作專區「共演短篇」大升級：生成過程不再怕中斷——任務進度會保存，就算服務重啟也能自動接續；生成時看得到進度、寫好的段落會先交付，完成時會通知你。寫完的短篇可以匯出 Markdown、TXT 或 EPUB，也能做成分享圖卡；新增書架把作品集中陳列，版本面板可以預覽舊版並一鍵還原。
2. 共演短篇更容易開始：一位角色就能開寫（第二位可選）；完成頁新增「接著做什麼」出口——直接續寫番外、或從某段回憶、某個時刻展開新篇；也提供起手模板，不知道從哪開始時降低門檻。開寫前會顯示素材豐富度，素材偏薄時會溫和提醒（不會擋你）。
3. 供應商「測試」按鈕現在是真的測試：會實際連到你設定的服務逐項檢查並回報——語言模型會列出模型清單並實測一句、嵌入模型會驗證維度是否相容、語音會列出聲音表或合成一小段、搜尋會真的查一次、圖片與影片服務會確認端點可達。API key 打錯、網址填錯、模型名稱不存在，按下測試幾秒內就會現形，不用等到聊天才發現。
4. 圖片供應商新增「深度測試」：真的生成一張小圖，驗證整條生圖鏈路都通（會產生一點點 API 費用，執行前會先跟你確認）。
5. 供應商設定修復：Base URL 欄位回來了——前一版因改版失誤消失，導致無法設定本地或自訂 API、「取得模型清單」一直報錯，編輯既有連線還可能弄丟已存的網址。內建服務（NanoGPT、OpenRouter 等）現在留空即用預設端點。
6. SearXNG 搜尋設定不再用猜的：欄位旁常駐說明現在會告訴你只要填實例根網址（例如 `https://searxng.example.com`），程式會自己組查詢路徑；也提醒實例需在 settings.yml 開啟 json 格式。之前存過的 SearXNG 設定不受影響，編輯儲存會自動轉換。
7. 圖片生成遇到儲存服務斷線時，不再回一句看不懂的系統錯誤：現在會明確告訴你「媒體儲存服務連不上，請檢查 storage 服務是否在跑」，後台日誌也會記下完整原因；重試時已生成的候選圖不會弄丟。
8. 傳圖給角色的對話更自然：修正了使用純文字聊天模型＋獨立看圖模型時，角色偶爾會把「圖片裡有些小字看不清」誤當成「你的訊息很難讀」來回應的問題；角色現在也分得清哪張圖是你剛傳的、哪張是稍早的、哪張是他自己生成的。
9. 用量與成本報表的入口直接出現在後台選單和首頁了，不用再手動輸入網址。
10. 「同場提示」可以關了：如果不想在打開聊天時看到「先傳訊息比較自然」的引導條，到「設定 → 個人 → 同場提示」關掉即可。關掉後只有你自己嘗試進入同場時才會顯示（該有的操作入口都還在）。
11. 從原始碼 build 的小修正：`docker compose build` 不再偶發 `image already exists` 錯誤（首次 clone 冷 build 最容易遇到）。
12. 模型相容性自癒：部分新世代模型會拒收舊參數、限制串流、或不接受系統訊息——現在系統會依伺服器的回覆自動調整並記住該模型的癖好，聊天不再因此中斷；各內建供應商的預設模型也全面更新到現行版本。
