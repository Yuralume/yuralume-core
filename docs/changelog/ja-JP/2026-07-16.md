# 2026.07.16 アップデート

1. 創作スタジオの「共演短編」が大きく進化：生成が中断に強くなりました——進行状況は保存され、サービスが再起動しても自動で続きから再開します。進捗が見え、書き上がったセクションから順に届き、完成時には通知が来ます。完成した短編は Markdown・TXT・EPUB に書き出せて、シェアカードにもできます。新しい本棚に作品をまとめて並べられ、バージョンパネルでは旧版のプレビューとワンクリック復元ができます。
2. 共演短編がもっと始めやすく：キャラクター一人から書けます（二人目は任意）。完成ページに「次にやること」の出口が増えました——そのまま番外編を続けて書く、あるいは思い出やあの瞬間から新しい一編を始める。書き出しに迷ったときのスターターテンプレートも用意しました。執筆前には素材の充実度が表示され、薄いときはやさしくお知らせします（止めはしません）。
3. プロバイダの「テスト」ボタンが本当にテストするようになりました：設定したサービスに実際に接続し、項目ごとに結果を報告します——LLM はモデル一覧の取得と 1 トークンの実対話、埋め込みモデルは次元の互換性検証、音声はボイス一覧の取得か短いサンプル合成、検索は実際に 1 回検索、画像・動画はエンドポイント到達性の確認。API キーの打ち間違い、URL の誤り、存在しないモデル名は、テストを押して数秒で分かります——会話の途中で気づくのではなく。
4. 画像プロバイダに「ディープテスト」を追加：実際に小さな画像を 1 枚生成して、生成チェーン全体が通ることを検証します（ごくわずかな API 費用が発生するため、実行前に確認します）。
5. プロバイダ設定の修正：Base URL 欄が戻りました——前リリースの改修ミスで消えており、ローカル/カスタム API が設定できない、「モデル一覧の取得」が常に失敗する、既存接続の編集で保存済み URL が消えることがある、という問題の原因でした。組み込みプリセット（NanoGPT、OpenRouter など）は空欄のままで既定エンドポイントを使います。
6. SearXNG の設定はもう推測不要：欄の横に常時表示されるヒントが、インスタンスのルート URL（例 `https://searxng.example.com`）だけを入れればよいこと（クエリパスはアプリが組み立てます）、インスタンス側の settings.yml で json フォーマットを有効にする必要があることを案内します。保存済みの SearXNG 設定はそのまま動き、次の保存時に自動変換されます。
7. 画像生成でストレージ障害が起きたとき、意味不明なシステムエラーは返しません：メディアストレージに接続できないこと、何を確認すべきかを明確に伝え、サーバーログには完全な原因が記録されます。リトライしても生成済みの候補画像は失われません。
8. キャラクターへの写真送信がより自然に：テキスト専用チャットモデル＋別の画像認識モデル構成で、「画像内の小さな文字が読み取れない」をキャラクターが「あなたのメッセージが読みにくい」と取り違えることがある問題を修正しました。どの画像がいま送られたものか、以前のものか、自分が生成したものかも正しく区別します。
9. 使用量・コストレポートの入り口が管理画面のメニューとホームに表示されるようになりました。URL を手打ちする必要はもうありません。
10. 「シーンヒント」をオフにできます：チャットを開いたときに「まずメッセージのほうが自然です」の案内が出るのが不要なら、「設定 → 個人 → シーンヒント」でオフに。オフでも、自分からシーンに入ろうとしたときだけは表示されます（必要な操作はすべて残ります）。
11. ソースからのビルドの小さな修正：`docker compose build` が `image already exists` で散発的に失敗しなくなりました（初回クローンのコールドビルドで起きやすかったものです）。
12. モデル互換性の自己修復：一部の新世代モデルは旧パラメータの拒否・ストリーミング制限・システムメッセージ非対応などがありますが、サーバーの応答に基づいて自動的に調整し、そのモデルの癖を記憶して会話を途切れさせません。組み込みプロバイダの既定モデルも現行版に一新しました。
