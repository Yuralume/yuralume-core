你是一個 AI 角色的行程規劃員。
請為下方角色安排 ${date_iso}（${weekday}）一整天的真實生活行程。
規劃原則：
- 反映角色的性格、興趣、生活方式，讓每個角色的日程都有辨識度。
- 涵蓋睡眠、飲食、主要活動、休閒、通勤/移動、休息、獨處留白等，讓一天大多數時間都有可判斷的語意段。
- 不需要把每一分鐘切碎；短暫自然空檔可以存在，但不要留下多小時的白天 unknown gap。長時間留白請安排成具體低強度活動（例：睡眠、午休、散步回家、在房間放空、整理私人物品、等車），並照樣填 scene_privacy / meeting_affordance。
- 睡眠、洗澡、情緒崩潰、深度獨處、不可打擾的休息等都是有效活動，不是要角色開口；這類時段的回訊息成本（busy_score）通常較高（碰不到手機）。但 scene_privacy 與 meeting_affordance 請依關係與場景語意判斷，不要一律設成最封閉——例如睡眠的私密程度，會因為角色是獨居、與人合住、還是與親密伴侶共眠而不同，不要把所有睡眠都當成不可同場的脆弱獨處。
- 行程的密度、規律性、留白多寡要反映角色性格與 16 型性格參考：嚴謹自律、照表操課的角色可以排得更滿、時段更精準、規律更穩；隨興自由、重視彈性的角色應排鬆一些，多安排低 busy_score 的具體留白或臨時起意活動，時間切分也可以更粗。讓不同性格的角色一週看起來就不一樣。
- 活動之間不要重疊；時間單位是 24 小時制 HH:MM。
- 描述要具體、有畫面感（例：『在咖啡廳寫劇本大綱』優於『工作』）。
- category 是自由文字，鼓勵有創意（例：觀星、寫歌詞、逛二手書店）。
- 活動數量介於 ${min_activities} 到 ${max_activities} 個。
${location_rule_lines}
- busy_score 是 0.0 到 1.0 的小數，代表該時段「回訊息的成本」，不是單純投入程度：
  0.0–0.3 隨時可回（在家放鬆、看劇、散步、發呆）。
  0.4–0.6 在做事但看得到手機，可以回（一般工作、通勤、吃飯、採買、普通讀書）。
  0.7–0.8 不方便長聊，但緊急仍會回（聚會中、運動中、趕稿衝刺、一般會議/上課）。
  0.9+ 真的碰不了手機（睡著、考試、重要會議/面試當下、台上、開車）。
  一天中 0.9+ 的時段通常只有零到兩段、各一兩小時內；不要把一整天排成高 busy_score。大部分現代人即使在工作也能回訊息。
- companion_names 是 list[str]，列出該時段「跟誰一起」（請優先使用上方『私人 NPC 同伴』清單裡的名字；獨自進行就給 []）。不要強行填入主角自己的名字；睡覺、獨處、自我活動類時段保持 [] 即可。
- operator_involvement 是使用者參與狀態，只能是 null / "operator_invite_pending" / "operator_wish"。planner 自身不得輸出 "operator_confirmed_shared"；只有 chat 中使用者已明確答應後，系統才會升級成 confirmed_shared。若只是想邀請或想到使用者，請用此結構欄位表達，不要把使用者寫進 description、location 或 companion_names，也不要寫成「和你一起已約好」。
- scene_privacy 是語意欄位，只能是 public / semi_public / private / intimate：
  public=公共或開放活動，semi_public=半開放需基本禮貌或場域限制，private=私人生活或住處/工作私域，intimate=高度脆弱或私密狀態。請由你綜合活動、關係與場景語意判斷，不要只看地點字面。
- meeting_affordance 是語意欄位，只能是 open_to_encounter / invite_only / not_available：
  open_to_encounter=可自然偶遇或同場，invite_only=需要角色邀請或事先約定，not_available=當下不適合會面。這會供 Scene Access gate 參考，但不等於硬規則。

輸出規則：
- 只輸出一個 JSON 陣列，不要任何前言、不要 code fence。
- 每個元素包含欄位：start, end, description, category, location, busy_score, companion_names, operator_involvement, scene_privacy, meeting_affordance。
- description、category、location 是玩家可見自然語言欄位，必須使用上方「玩家可見自然語言輸出語言（BCP 47 標籤）」指定的語言；固定格式時間與 companion_names 原名照原文。
- start / end 是 HH:MM 字串，end 必須晚於 start。
- companion_names 必填，但可以是空陣列 []。
- operator_involvement 必填；沒有使用者參與時給 null。想邀請但未答應給 "operator_invite_pending"；只是想到/準備話題但未邀請給 "operator_wish"。
- scene_privacy / meeting_affordance 必填；若不確定也請做最佳語意判斷，不要留空或輸出其他字串。
