- 若沒有明確證據，優先保持 active。
- 只有在對話明顯顯示目標已達成時才給 done。

輸出規則：
- 只輸出一個 JSON 物件，不要任何前言或 code fence。
- 物件包含兩個欄位：verdicts 與 new_goals。

verdicts 欄位：
- JSON 陣列，每個元素包含 id, status, notes。
- id 必須出自上方目標清單（若不在清單中，該項會被忽略）。
- status 必須是以下其中之一：${status_hint}。
- notes 是不超過一句話、描述判斷理由；此欄位玩家可見，必須使用上方指定的玩家可見輸出語言；可選。
- 若某目標沒有變化，省略該目標即可（不要重複列出）。

new_goals 欄位：
- JSON 陣列，每個元素包含 content, priority, tags。
- content 是一句話描述角色接下來想推進的事（第一人稱或目標式）；此欄位玩家可見，必須使用上方指定的玩家可見輸出語言。
- priority 是 1-5 的整數，3 為預設，5 最重要。
- tags 是最多 5 個小寫標籤。
- 最多 ${max_new_goals} 筆；若沒有必要的新目標，給空陣列 []。

範例輸出：
{"verdicts": [{"id": "abc-123", "status": "done", "notes": "使用者已完成吉他社發表"}], "new_goals": [{"content": "陪使用者準備下次發表的曲目", "priority": 3, "tags": ["music"]}]}
